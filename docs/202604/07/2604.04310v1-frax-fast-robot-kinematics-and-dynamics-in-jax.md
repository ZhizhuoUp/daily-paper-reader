## 摘要
在机器人控制、规划和学习中，需要高性能、易于使用且兼容 CPU 和加速器的刚体动力学库。虽然现有库通常在低延迟 CPU 执行或高吞吐量 GPU 工作负载方面表现出色，但很少有库能提供一个针对多种架构的统一框架，且不牺牲性能或易用性。为解决这一问题，我们推出了 frax，这是一个基于 JAX 的机器人运动学与动力学库，在 CPU、GPU 和 TPU 上提供高性能的纯 Python 接口。通过全向量化的机器人动力学方法，frax 实现了高效的实时控制和并行化，同时支持基于优化方法的自动微分。在 CPU 上，frax 实现了微秒级的计算时间，适用于千赫兹级别的控制频率，性能优于常见的 Python 库，并接近经过优化的 C++ 实现。在 GPU 上，同样的代码可以扩展到数千个实例，每秒可进行超过 1 亿次动力学评估。我们在 Franka Panda 机械臂和 Unitree G1 人形机器人上验证了性能，并将 frax 作为开源库发布。

## Abstract
In robot control, planning, and learning, there is a need for rigid-body dynamics libraries that are highly performant, easy to use, and compatible with CPUs and accelerators. While existing libraries often excel at either low-latency CPU execution or high-throughput GPU workloads, few provide a unified framework that targets multiple architectures without compromising performance or ease-of-use. To address this, we introduce frax, a JAX-based library for robot kinematics and dynamics, providing a high-performance, pure-Python interface across CPU, GPU, and TPU. Via a fully-vectorized approach to robot dynamics, frax enables efficient real-time control and parallelization, while supporting automatic differentiation for optimization-based methods. On CPU, frax achieves low-microsecond computation times suitable for kilohertz control rates, outperforming common libraries in Python and approaching optimized C++ implementations. On GPU, the same code scales to thousands of instances, reaching upwards of 100 million dynamics evaluations per second. We validate performance on a Franka Panda manipulator and a Unitree G1 humanoid, and release frax as an open-source library.

在机器人领域，高性能的刚体动力学计算是控制、规划和强化学习的核心。现有的动力学库（如 Pinocchio, MuJoCo）面临“性能鸿沟”：
*   **CPU 库**：擅长低延迟的单实例实时控制，但难以扩展到 GPU 上的大规模并行任务。
*   **GPU 库**：擅长高吞吐量的模拟（如强化学习训练），但在 CPU 上的单实例执行效率较低。
*   **集成困难**：许多库使用 C++ 编写，其 Python 接口存在调用开销，且难以与现代自动微分（AD）框架（如 JAX）无缝集成。
**frax 的目标**是提供一个纯 Python、基于 JAX 的统一框架，在保持 CPU 低延迟的同时，实现 GPU 的高吞吐量，并原生支持自动微分。

#### 2. 方法论：核心思想与关键技术
frax 的核心在于**全向量化（Fully-Vectorized）**的动力学算法实现：
*   **核心思想**：放弃传统的递归树遍历（通常为 $O(n)$ 复杂度），转而采用基于矩阵运算的向量化表述。
*   **祖先掩码（Ancestor Mask $U$）**：预计算一个 $n \times n$ 的下三角矩阵 $U$，其中 $U_{ij}=1$ 表示关节 $j$ 是关节 $i$ 的祖先。通过 $U$ 与空间代数向量的矩阵乘法，将树状结构的求和运算转化为高效的线性代数运算。
*   **算法改进**：
    *   **RNEA（递归牛顿-欧拉算法）**：通过向量化将复杂度从 $O(n)$ 变为 $O(n^2)$。
    *   **CRBA（复合刚体算法）**：将复杂度从 $O(nd)$ 变为 $O(n^2)$。
*   **权衡逻辑**：虽然理论复杂度增加，但这种设计消除了 Python 循环，极大地利于 XLA 编译器的算子融合（Operator Fusion）和硬件并行，同时简化了自动微分的计算图追踪。
*   **JAX 集成**：利用 `jax.jit` 进行即时编译，利用 `jax.jvp`（雅可比向量积）直接计算控制律所需的导数，避免了显式计算全雅可比矩阵的开销。

#### 6. 主要结论与发现

*   **编译效率**：frax 的 JIT 编译时间显著短于 MJX/BRAX（1-2 秒 vs 6-12 秒），提升了开发调试效率。

#### 7. 优点
*   **跨平台一致性**：同一套 Python 代码可无缝运行在 CPU、GPU 和 TPU 上。
*   **高性能与易用性兼得**：既有 Python 的开发便捷性，又通过向量化和 XLA 编译获得了接近 C++ 的性能。
*   **原生支持自动微分**：为基于优化的控制和端到端学习提供了极大的便利。

#### 8. 不足与局限
*   **功能覆盖有限**：目前仅支持球体和平面碰撞模型，尚不支持复杂的网格或凸包碰撞。
*   **算法完整性**：尚未实现正向动力学（ABA）和质量矩阵的解析导数，目前依赖自动微分。
*   **复杂度风险**：对于自由度极高（如 $n > 100$）的系统，$O(n^2)$ 的向量化开销可能会超过其带来的并行收益。
*   **应用限制**：主要针对刚体动力学，不涉及软体或复杂流体交互。
