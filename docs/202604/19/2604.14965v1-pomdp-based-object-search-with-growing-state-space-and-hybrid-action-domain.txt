Abstract 

Efficiently locating target objects in complex indoor environments with diverse furniture, such as shelves, tables, and beds, is a significant challenge for mobile robots. This difficulty arises from factors like localization errors, limited fields of view, and visual occlusion. We address this by framing the object-search task as a high-dimensional Partially Observable Markov Decision Process (POMDP) with a growing state space and hybrid (continuous and discrete) action spaces in 3D environments. Based on a meticulously designed perception module, a novel online POMDP solver named the growing neural process filtered k-center clustering tree (GNPF-kCT) is proposed to tackle this problem. Optimal actions are selected using Monte Carlo Tree Search (MCTS) with belief tree reuse for growing state space, a neural process network to filter useless primitive actions, and k-center clustering hypersphere discretization for efficient refinement of high-dimensional action spaces. A modified upper-confidence bound (UCB), informed by belief differences and action value functions within cells of estimated diameters, guides MCTS expansion. Theoretical analysis validates the convergence and performance potential of our method. To address scenarios with limited information or rewards, we also introduce a guessed target object with a grid-world model as a key strategy to enhance search efficiency. Extensive Gazebo simulations with Fetch and Stretch robots demonstrate faster and more reliable target localization than POMDP-based baselines and state-of-the-art (SOTA) non-POMDP-based solvers, especially large language model (LLM) based methods, in object search under the same computational constraints and perception systems. Real-world tests in office environments confirm the practical applicability of our approach. Project page: https://sites.google.com/view/gnpfkct .

## 1 Introduction 

Object search in partially familiar, cluttered environments is crucial for mobile robots. For example, a robot retrieving a customer’s order from a warehouse or fetching a Coke bottle from a kitchen must often locate a target object that is partially or fully obscured by other items. Known as the ‘Mechanical Search’ problem (Danielczuk et al., 2003), it typically involves leveraging partial environmental knowledge, like maps of large furniture (e.g., shelves, tables), while smaller objects (e.g., mugs, glasses) remain unlocalized due to frequent movement. Humans typically find smaller objects by relying on prior knowledge, assuming familiarity with the target, and removing obstructions as needed (Aydemir et al., 2013). Similarly, robots must adjust their configurations, strategically observe from multiple viewpoints, and interact with the environment by removing obstructions and retrieving the target object. This paper focuses on object search in a multi-room home environment with predefined workspaces. Equipped with 3D point cloud and 2D occupancy grid maps and provided with target photos, the robot aims to efficiently locate objects using onboard sensors, as shown in Fig. 1. ## 2 Related work 

## 2.1 Mechanical search: Planning 

Robots frequently encounter challenges navigating cluttered environments to locate target objects. Many contemporary approaches leverage advance-ments in learning technologies, including deep rein-forcement learning (RL) (Kurenkov et al., 2020), deep Q-learning (Yang et al., 2020), and deep-geometric inference systems (Huang et al., 2021). For instance, (Kurenkov et al., 2020) proposes an innovative learning procedure that combines an asymmetric architecture to exploit privileged information, guidance from suboptimal teachers, and mid-level representations for training deep RL agents to discover target objects hidden within piles of unknown items. However, these methods are tailored to specific environments, such as shelves or boxes, limiting their applicability in diverse, unstructured settings. Beyond learning-based methods, the POMDP framework offers a compact and effective approach for object search. As an example, in (Huang et al., 2022), a Monte-Carlo tree search method featuring a 1D occupancy distribution for objects is employed to swiftly identify and extract the target object. Due to its natural compatibility with scenarios involving multiple detected objects, the object-oriented POMDP (OO-POMDP) framework is widely adopted. This framework factors objects into classes with associated attributes, treating the beliefs for different objects as independent, which enables linear scaling of belief size and reduces

For example, (Aydemir et al., 2013) proposed a 2D POMDP formulation for object search in a room by evaluating candidate viewpoints. Recent efforts have extended this to 3D environments with object-size-friendly properties using multi-resolution planning algorithms (Zheng et al., 2021). ## 2.2 Mechanical search: Reasoning 

Earlier robotics approaches integrated reasoning directly into the planning process, such as updating probabilities, rather than treating it as a distinct module. As robotic systems became more sophisticated, reasoning evolved into aseparate layer, addressing high-level cognitive tasks like commonsense inference, contextual understanding, and hypothesis generation. This reasoning layer then informs a planning module to execute detailed action strategies. For instance, (Giuliari et al., 2023) demonstrates a reasoning module that uses environmental context and commonsense knowledge to infer plausible object locations, aiding localization in partially observed scenes. Similarly, (Ge et al., 2024) employs commonsense knowledge from large language models to construct scene graphs for improving object search. However, these methods struggle with unconventional object arrangements, such as randomized setups. In our scenario, objects are deliberately placed in configurations that defy typical human habits, increasing search complexity and requiring active robot-environment interactions, such as occlusion removal. Our reasoning idea is based on the probability update of the grid world. By representing the belief over the pose state of each object in the planning environment using particle filtering, the authors in (Garrett et al., 2020) incorporate probabilistic reasoning into a deterministic planner and then perform replanning when facing the base movement failure. Recent work (Huang et al., 2024) utilizes a video tracking-based memory model with integrated reasoning and planning capabilities. This allows the system to remember potential locations of occluded target objects and complete tasks using appropriate actions. 

## 2.3 POMDP formulation and solvers 

For a POMDP agent, the process commences with a brief, where the agent deduces and executes the optimal action to transition to anew state. This new state is concealed within an updated belief, refined through Bayesian inference based on observed data. Each step yields an immediate reward and contributes to a discounted cumulative reward, fostering a long-term objective. This iterative process continues until terminal conditions are met. The POMDP framework is a powerful tool for planning under uncertainty, excelling in non-deterministic and partially observable scenarios. Widely applied in robotics, it addresses tasks like autonomous underwater vehicle navigation (Hou et al., 2021), robot manipulation (Pajarinen et al., 2019), and Human-Robot collaboration (Burks et al., 2023). Our work adopts this approach, developing an object-oriented POMDP model with hybrid action domains. Finding the exact optimal strategy for a POMDP problem is widely recognized as computationally intractable (Papadimitriou et al., 1987). Over the past two decades, POMDP solvers have been categorized into offline and online methods. Offline solvers, such as point-based POMDP methods (Shani et al., 2013,K), can produce optimal solutions for small-scale problems. However, their significant memory requirements often make them impractical for use on standard computational platforms in real-world applications. In contrast, online solvers, especially sampling-based methods, offer a practical trade-off by achieving approximate optimality with greater computational efficiency, making them suitable for various applications. Sampling-based solvers, such as POMCP (Silver et al., 2010), adaptive belief tree (ABT) (Kurniawati et al., 2016), and determinized sparse partially observable tree (DESPOT) (Somani et al., 2013), share a common strategy: representing belief as particles and utilizing Monte Carlo tree search (MCTS) to expand the belief tree within computational limits. Despite significant advancements, solving POMDPs with high-dimensional continuous action spaces remains a substantial challenge. The key point in existing continuous-action POMDPs is to refine the action subset incrementally to improve the possibility that the selected subset of actions contains the best action. Methods like partially observable Monte Carlo planning with observation widening (POMCPOW) (Sunberg et al., 2018) use Progressive Widening to add new sampled actions after sufficient exploration, while others integrate

## 2.4 Neural process in robotics 

As a more efficient alternative to Gaussian processes, neural processes excel in function regression by capturing uncertainty in predictions as a stochastic process. In numerous robotics tasks, the ability to account for uncertainty in predictions is crucial for enhancing system robustness and practicality. For instance, (Chen et al., 2022) introduces a conditional neural process (CNP)-based meta-learning algorithm to estimate grasp points from limited depth image trials. The versatility of CNPs is further demonstrated in applications such as 6D pose estimation (Li et al., 2022) and understanding social aspects of navigation (Yildirim et al., 2022). In our work, we utilize neural processes to address uncertain scoring function regression and primitive action filtering, creating a robust framework for decision-making in complex, dynamic environments. This idea was publicly released in our earlier work (Chen et al., 2024b), preceding several concurrent studies exploring neural process-based planning (Jain et al., 2025). 

## 3 Overall framework 

In a cluttered indoor environment with numerous unknown movable and immovable obstacles, arobot must locate and pick up a movable target object using onboard sensors and reference photos. To solve this, we use a POMDP-based approach, GNPF-kCT, with 4 alternating stages: planning, execution, observation, and filtering. In the planning stage, a look-ahead MCTS exploration uses an initial belief, transition, and reward functions, observation model, rollout strategy, and terminal function to sample particles and determine the best action. To address challenges from a growing state space and continuous action domain, we enhance the POMDP solver with belief tree reuse, neural process filtering, and k-center clustering, improving efficiency and performance. At the execution and observation stages, the algorithm performs the selected action on the robot, which gathers real-world measurements from onboard sensors, including 2D lidar scans, 3D point clouds, and RGBD camera images, without relying on markers like QR codes. These measurements are processed into state-space information through steps like robot pose estimation, point cloud filtering and segmentation, principal component analysis, object matching, and move-ability estimation. In the filtering stage, a particle filter updates the belief and prunes the belief tree using processed sensor data. The guessed target object and its grid world are updated via log-odds, similar to occupancy grid mapping. Robot pose estimation uses iterative closest point (ICP), initialized by adaptive Monte Carlo localization (AMCL). If the terminal condition is met, the task ends with the cumulative reward; otherwise, the updated belief reuses the existing belief tree and resamples particles to proceed with the MCTS. These steps, as shown in Fig. 2, collectively achieve object search and retrieval.

We also have the point cloud and occupancy grid maps for a planning environment with furniture for robot navigation. While point clouds for some known obstacle objects are available, the target object and many other obstacles lack prior point cloud data, requiring detection via onboard sensors during execution. Unlike the assumption in OO-POMDP, because we measure the information about the object using real-world sensors, the object beliefs in our scenario are not independent and may have effects on each other. ## 5 Online POMDP solver: GNPF-kCT 

## 5.1 Overview 

GNPF-kCT is an anytime online POMDP solver that assumes the Q-value of the POMDP prob-lem follows Lipschitz continuity in the action space ¶. The solver alternates between planning, execution, observation, and filtering, focusing on planning to identify the optimal action from the current belief b0. To reduce the complexity of the high-dimensional continuous action domain, a pre-trained neural process predicts the feasibility of sampled actions with associated uncertainties, fil-tering out irrelevant actions. The remaining actions are clustered into hyperspheres using k-center clustering. A belief tree T is then constructed, where nodes represent beliefs and actions. Each belief node node o ∈ T is linked to a dynamic list 

L(node o), initialized from the previous tree and augmented with newly generated hyperspheres to include connected action nodes. During episode simulations, L(node o) expands as k-center clus-tering refines the action space further. In POMDP problems with growing state spaces (e.g., detecting new objects), the observation introduces unpre-dicted states, requiring classical solvers to rebuild the entire belief tree, which is computationally expensive. GNPF-kCT avoids this by saving a his-tory tuple ¯H =< H s, H r , H, dim r (Hs) >. The list 

Hs tracks all state histories from simulations, where rows represent particle IDs and columns track their state histories. Two additional lists, H (action-observation history) and Hr (immediate rewards), as well as a vector dim r (Hs) (depth vector) are associated with Hs. These saved histories allow the solver to approximate and reuse the previous belief tree by updating Hs, avoiding a full rebuild. Algorithm 1 outlines the complete method, with details provided in subsequent sections ‖.

## 5.2 GNPF-kCT: NP network for filtering 

5.2.1 Motivation about network filtering. In com-plex tasks, many primitive actions with contin-uous control parameters (Ac) are often ineffi-cient or impractical for achieving long-term goals, such as the optimal policy goal of POMDPs, or short-term goals, like completing a primitive action. Specifically, if the optimal action a∗ =

argmax a∈A c Q(b, a) lies within the continuous action domain Ac, there exists a smaller feasible region X ⊆ A c that satisfies a∗ ∈ X . An intuitive idea is to use a score function g, with some inherent uncertainty, to quickly identify this region X given 

s and α(P), where α(P) represents all configura-tion settings of the POMDP problem P. Gaussian Processes (GPs) are a natural choice for modeling such functions, leveraging probabilistic inference to update predictions, but they become com-putationally expensive in high-dimensional tasks. To address this, we propose to use Neural Pro-cesses (NPs), which also define distributions over functions while being computationally efficient and adaptable to various inputs leveraging the advantages of neural networks. Unlike traditional approaches like Q-learning (Hausknecht et al., 2015), which train complex networks with large minimum description lengths (Zhao et al., 2024a) to pursue X = a∗, we adopt a simpler scoring net-work. This network prioritizes short-term, physi-cally meaningful goals (e.g., single-step objectives), making training and implementation more practi-cal. The NP-based scoring network efficiently filters out ineffective actions, enabling a more compact POMDP framework to select optimal actions based on the filtered actions and current beliefs. We employ attentive NPs nn (μ(a), σ (a)) to learn the scoring function, where μ(a) is the mean func-tion and σ(a) is the kernel function. Importantly, our aim is not to mandate the use of the same network for all POMDP tasks but to highlight the action-filtering concept. Users are encouraged to train domain-specific networks tailored to their problems, integrating short-term primitive action learning with long-term POMDP frameworks in continuous action domains:

## 5.3 GNPF-kCT: Belief Tree Construction 

The belief tree T is a tree of which the nodes include the belief nodes node o and action nodes 

node a and the edges are action a ∈ A and observation o ∈ O . The belief nodes node o save a set of states s ∈ b forming belief b. An action node node a is a child of the belief node node o

via edge a if and only if transition function 

T (s, a, s′) satisfies. Belief nodes are linked by the action–observation pairs ( a, o) based on transition function T (s, a, s′) and observation function 

Z(s′, a, o). The episode simulation framework is in Algorithm 3. 

5.3.1 Construction overview. To construct the belief tree T , our GNPF-kCT framework follows the standard select-expand-simulate-backup strat-egy used in many MCTS algorithms, with an adap-tive discretization mechanism based on the k-center clustering method for controllable discretization rates. As mentioned before, each belief node main-tains a dynamic list L(node o). If resources allow, episodes are continuously sampled to grow the belief tree T , at each episode, we select a path from the tree root with the format: s0, a0, o0, r0, s1, a1,

o1, r1, · · · . Starting from the root belief, an action 

ai ∈ L (node o) is selected using a revised UCB1 strategy (based on HOO, detailed in Section 5.3.2).

H(center ∗, range ∗) and then sample a discrete action in this hypersphere, similar to ellipsoid sampling. 

5.3.4 Backup. When each episode reaches the terminal state, our GNPF-kCT framework updates the estimation reward ˆQ(node o, a) as well as the visited numbers N (node o) and N (node o, a)of all nodes visited by this episode. Here, we present two classical stochastic backup methods including the Bellman backup (Algorithm 6), which is used in the ABT method and similar to the rule used in Q-learning, and the Monte-Carlo backup (Algorithm 6), which is widely used in many outstanding POMDP solvers, like POMCP, POMCPOW, and Voronoi Optimistic Monte Carlo Planning with Observation Weighting (VOMCPOW) (Lim et al., 2021). The Bellman update naturally follows the objective function of the POMDP formulation that aims to pursue optimal action in each step of the long-term planning. It helps the solver to explore deeper by focusing its search on promising parts of the belief tree. The main challenge for the Bellman backup is when facing unexpected observations, a lot of deeply explored belief trees will be frequently cut and this case causes poor planning performance. Hence, the Bellman backup gets a better performance when the good rewards are sparse in the belief tree, but it is not stable enough for the poor observation prediction. In contrast to selecting the reward with optimal action, the Monte-Carlo backup computes the average reward along with different action episodes, which means that the generated belief tree will be more robust when facing unexpected uncertainty in received observation. In our application for object search, the real visual observation is not well predictable for the observation model in the POMDP formulation and the robot camera will frequently receive unexpected measurements, which breaks the advantage of the Bellman update. We use the Monte-Carlo backup in our problem but the Bellman backup is commonly superior in other applications, so we present both two backup ways here.

State-of-the-art real-time object detection systems like YOLO classify objects into predefined categories, which may not match specific targets. Meanwhile, 3D point clouds provide valuable data for image segmentation. To tackle object detection with limited images and optional semantic cues, we combine traditional feature matching with YOLO, leveraging their strengths for accurate and efficient detection. Building on point cloud segmentation, we project segmented local frame point clouds onto the image using the camera configuration, creating sub-images. YOLOv5 (Jocher et al., 2020) generates sub-images with semantic scores 

{syi , i = 1, 2, · · · , m}, where m̸ = n typically. A nearest-neighbor-based method aligns sub-images by matching those with sufficient overlap. Matched pairs link local sub-images to objects, and SIFT descriptors match these to the target object. Detection probability is defined as the ratio of matched features to total features, 

{sdi , i = 1, 2, · · · , n }. For specified target types (e.g., a cup), semantic scores syi and detection probabilities sdi are averaged. As real-world object matching can lack robustness and perception is not our main focus, especially for uniform-colored objects (e.g., a red bottle), we add color matching. ## 6.4 Move-ability estimation 

In real-world settings, some objects may be immovable due to size, manipulator limits, or base motion constraints. Our framework focuses on clearing the FOV by manipulating objects, requiring the estimation of movability probabilities and updating these beliefs within the POMDP planning. Using point cloud segmentation on the fused global cloud, we isolate objects and use the Grasp Pose Detection (GPD) toolbox (Ten et al., 2017) to predict multiple grasp poses. To reduce complexity, k-means clustering selects k diverse, high-success poses pgi , i = 1 , 2, · · · , k . These poses and nearby obstacles are transformed into local frames T gr (pgi ) based on pre-visited robot poses 

pri , ensuring feasibility. The transformed poses are used as planning targets for the robot manipulator via the moveit toolbox, within a time limit tm,but without execution. Feedback from moveit determines the movability probability 0 .5 < m oi ≤

1. If no solution is found, moi = 0 .5 ‖, and objects above a size threshold are non-movable (moi = 0). During planning, movability is assessed by sampling and comparing with moi . Fig. 11 illustrates candidate grasp poses (blue forks).

## 7.3 Comparison with non-POMDP framework 

All previous results were obtained using our pro-posed POMDP framework. In this section, we com-pare our method with several representative non-POMDP planning frameworks, including Random, NPF-G, SGoLAM (Kim et al., 2021), SayPlan (Rana et al., 2023), and MoMa-LLM (Honerkamp et al., 2024). Since these planning methods differ substantially in their original problem formulations and configuration settings, we adapt them to our task by minimally revising their action spaces and unifying the perception module across all methods, ensuring a fair comparison. We first provide aqualitative summary of the original capabilities of these methods, as reported in the corresponding references, in Table 4. As shown in Table 4, our framework is imple-mented in the high-fidelity Gazebo environment and operates with minimal prior information about the scene. The entire pipeline, including percep-tion and manipulation actions, is designed to be directly transferable to real robotic systems with very little modification. In contrast, LLM-based methods such as SayPlan and MoMa-LLM typ-ically rely on pre-given semantic annotations or simulator-provided ground-truth information. For example, MoMa-LLM directly accesses pixel-level semantic perception from the iGibson simulator and employs QR-code markers in real-world experi-ments, both of which are treated as uncertainty-free prior knowledge (marked as Pre-given in Table 4). These simplified assumptions make certain actions easier, for instance, exploration can be performed by a single rotation that immediately reveals all object labels. Moreover, SGoLAM and SayPlan do not support interactive actions that modify the environment configuration. While MoMa-LLM includes limited interactions such as opening doors or refrigerators, it does not allow interactive manip-ulation of detected objects. In contrast, our frame-work explicitly models environment interaction and uncertainty, enabling more realistic and transfer-able decision-making. We next detail the compared methods: 

Random This baseline adopts a random high-level policy that selects actions uniformly at random from both discrete and continuous action spaces. Specifically, we first sample a scalar γs ∼U(0 , 1). If γs > 0.3, a continuous action is selected, and its parameters are sampled uniformly from the predefined continuous ranges. If γs ≤ 0.3, we check whether any objects are available for declaration or removal. If such objects exist, one is randomly selected for declaration or removal; otherwise, a continuous action is selected as in the case of γs >

0.3. 

NPF-G This baseline employs a greedy high-level decision-making strategy over both discrete and continuous actions. Since object declaration and target-object removal consistently yield positive rewards, these actions are always selected whenever available. Otherwise, a scalar γs ∼ U (0 , 1) is sampled and compared with 0.1. If γs > 0.1, a continuous action is selected by choosing the one with the highest signal-to-noise ratio among actions predicted as efficient by the trained neural network, with a likelihood of at least 20%. If γs ≤ 0.1, the policy checks for available removal actions and randomly selects one if present; if none are available, it defaults to the same continuous action selection strategy used when γs > 0.1.

This limitation is closely related to the absence of an explicit internal world model in current LLMs, which hinders long-horizon action–outcome simulation. Overall, we believe that integrating model-based reasoning tailored to single-room physical interactions with learning-based global reasoning mechanisms represents a more promising research direction, and is likely to lead to more realistic and robust solutions for real-world object search tasks. 

## 7.4 Limitation 

Our approach faces several limitations. First, reliance on a pre-existing map is impractical for real-world robots, but this can be mitigated with advanced SLAM techniques. Second, achieving a 100% success rate for primitive actions in real-world scenarios is unrealistic, affecting overall performance. Additionally, our use of point cloud segmentation may generate incorrect bounding boxes for objects with large contact areas, leading to errors in data association and belief updates. Object detection methods like YOLO, SIFT, and color matching also face challenges in environments with limited features, low coverage, or poor lighting. But we believe these issues can be addressed through advances in perception, navigation, and execution, which are outside the scope of this paper. 

## 8 Conclusion and future work 

This paper introduces GNPF-kCT, a novel POMDP-based framework and solver for object search tasks in a 3D environment using a mobile robot manipulator. The problem is modeled as an 8-tuple POMDP, focusing on FOV adjustments and belief updates for observed objects. By incorporating a guessed target object, our approach efficiently locates hidden objects by removing obstacles, a limitation in existing methods. Our framework combines a pre-trained NP network to select potential primitive actions from Gazebo simulation data with adaptive k-clustering hypersphere partitioning to refine the continuous action space for efficient MCTS exploration. This integrates short-term learning with a compact long-term POMDP framework for improved performance. To address challenges from growing state spaces, we reuse the previous belief tree and apply a revised UCB1 bound, incorporating hypersphere range and belief distance for efficient and robust decision-making. Experiments show that, our solver surpasses classical methods in cumulative reward, task steps, and success

Notably, our framework requires only maps, photos, and onboard sensors, eliminating the need for geometric models or additional object-specific data, and making it highly practical for real-world applications. This paper lays the groundwork for autonomous object search. Future work will integrate advanced methods, such as LLMs (Zhao et al., 2024a) to embed expert knowledge via prompts (Tang et al., 2025), or world model (Yupeng et al., 2025) to fast predict system in latent space, and enable autonomous map creation and workspace generation. More importantly, in experimental results, we find that one of the main limitations is the success rate of the complicated primitive actions. Even though we can consider these uncertainties in the planning framework, they will still cause the low success rate of the whole system with poor preventive implementation. The well-signed Vision-Language-Action (VLA) method, like RT-2 (Zitkovich et al., 2023) and Octo (Team et al., 2024), may be helpful in completing better primitive actions. ## F Parameter selection 

Our solver relies on 8 parameters, including the refining clustering number k, coefficients ω1, ω2,and ω3 for the MCTS action selection strategy, a self-defined exploration constant Cr , the minimum radius for partitioning Dlim , and two coefficients controlling the refining velocity, with 0 ≤ ¯ω2 <

¯ω1 ≤ 1. Most of these parameters were not fine-tuned for optimal performance; instead, they were quickly identified or chosen intuitively. Here, we would like to add some explanations to help users quickly determine the parameters within a short time, like 20 minutes. Let’s go through each parameter: 

Refining Clustering Number k:Fig. 20 shows that k = 3 , 4, and 5 result in similar performance, indicating robustness and flexibility in selecting this parameter. For general POMDP problems, I recommend using the default value k = 3 (chosen arbitrarily before parameter experiments) or the optimal clustering value, obtained during the list initialization update before MCTS, which ki = 4 in our paper. If the value ki =4 obtained in the list initial update before MCTS is used, this parameter k becomes non-heuristic and consistently achieves good performance. 

Coefficients ω1, ω2, and ω3:These coefficients are straightforward to select. Without specific domain knowledge, I use the common value ω1 = √2, derived theoretically from the multi-armed bandit problem based on Hoeffding’s inequality. For ω2 and ω3, ensure that

• Run the POMDP problem for one step within the time limit and identify the number of their commonly used particles, like Np = 150.
