## 摘要
云机器人技术使机器人能够将高维运动规划和推理卸载到远程服务器。然而，对于需要高频控制的连续操作任务，网络延迟和抖动会严重破坏系统的稳定性，导致指令匮乏和不安全的物理执行。为解决这一问题，我们提出了投机性策略编排（Speculative Policy Orchestration, SPO），这是一种具有延迟韧性的云边框架。SPO 利用云端托管的世界模型预计算未来的运动学路标点，并将其流式传输到本地边缘缓冲区，从而将执行频率与网络往返时间解耦。为了减轻预测漂移导致的不安全执行，边缘节点采用了一个 ε-管道验证器（ε-tube verifier），严格限制运动学执行误差。该框架结合了自适应时界缩放机制，根据实时跟踪误差动态扩展或收缩投机性预取深度。我们在模拟网络延迟环境下，通过连续 RLBench 操作任务对 SPO 进行了评估。结果表明，即使部署精度适中的学习模型，与阻塞式远程推理相比，SPO 也能将网络引起的空闲时间减少 60% 以上。此外，与静态缓存基准相比，SPO 丢弃的云端预测减少了约 60%。最终，SPO 在保持有界物理安全性的同时，实现了流畅、实时的云机器人控制。

## Abstract
Cloud robotics enables robots to offload high-dimensional motion planning and reasoning to remote servers. However, for continuous manipulation tasks requiring high-frequency control, network latency and jitter can severely destabilize the system, causing command starvation and unsafe physical execution.   To address this, we propose Speculative Policy Orchestration (SPO), a latency-resilient cloud-edge framework. SPO utilizes a cloud-hosted world model to pre-compute and stream future kinematic waypoints to a local edge buffer, decoupling execution frequency from network round-trip time. To mitigate unsafe execution caused by predictive drift, the edge node employs an $ε$-tube verifier that strictly bounds kinematic execution errors. The framework is coupled with an Adaptive Horizon Scaling mechanism that dynamically expands or shrinks the speculative pre-fetch depth based on real-time tracking error.   We evaluate SPO on continuous RLBench manipulation tasks under emulated network delays. Results show that even when deployed with learned models of modest accuracy, SPO reduces network-induced idle time by over 60% compared to blocking remote inference. Furthermore, SPO discards approximately 60% fewer cloud predictions than static caching baselines. Ultimately, SPO enables fluid, real-time cloud-robotic control while maintaining bounded physical safety.

### 1. 核心问题与整体含义（研究动机和背景）
随着深度神经网络（如扩散策略、Transformer）在机器人操作中的应用，策略模型变得越来越大，通常需要卸载到云端执行。然而，高频连续控制（10-50 Hz）对网络延迟极其敏感。在现实的5G或Wi-Fi环境下，网络抖动和延迟（通常在100ms以上）会导致“指令匮乏”（Command Starvation），使机器人动作停顿、失稳甚至产生安全隐患。本文的研究动机是借鉴大语言模型（LLM）的“投机采样”和云游戏的“投机执行”技术，解决云机器人操作中的延迟韧性问题。

### 2. 方法论：核心思想与关键技术
SPO 框架的核心思想是将高维轨迹生成的频率与网络往返时间（RTT）解耦。
*   **云端投机生成（Cloud-Side Speculation）**：云端不仅运行策略模型（Policy），还运行一个世界模型（World Model）。它根据当前状态自回归地预测未来 $K$ 个步长的运动学路标点（Waypoints），形成一个“投机块”流式传输给边缘端。
*   **边缘端 $\epsilon$-管道验证器（$\epsilon$-tube Verifier）**：边缘端接收到预测轨迹后，在每个控制周期对比机器人实际状态与预测状态。通过计算归一化距离，若误差在 $\epsilon$ 阈值内则执行缓存指令；若超出阈值（发生碰撞或预测漂移），则立即清空缓存并触发安全停止。
*   **自适应视界缩放（Adaptive Horizon Scaling, AHS）**：这是一种反馈机制。系统根据实时跟踪误差动态调整投机深度 $K$。在平滑运动时增加 $K$ 以掩盖延迟；在接触频繁、不确定性高的阶段减小 $K$，以降低误执行风险并节省带宽。

### 3. 实验设计
*   **实验场景**：基于 **RLBench** 机器人学习基准测试，选择了三个难度递增的任务：

    2.  **Insert Onto Square Peg**（套圈）：中等复杂度，涉及精密对准。
    3.  **Put All Groceries In Cupboard**（放杂物进橱柜）：高复杂度，涉及多物体和遮挡。
*   **网络环境**：模拟广域网（WAN）环境，设定 150ms 的基础延迟和 ±30ms 的随机抖动。
*   **对比方法（Baselines）**：

    *   **T1-SC**：仅预取 1 步的简单投机。

### 6. 主要结论与发现
*   **显著降低延迟影响**：在 150ms 延迟下，SPO 比阻塞式远程推理减少了 **60% 以上**的网络诱导空闲时间。

*   **安全性与流畅性兼顾**：$\epsilon$-tube 验证器能有效捕获物理接触导致的偏差，防止开环执行带来的危险，同时保持了动作的连贯性。

### 7. 优点
*   **模型无关性**：SPO 作为一个编排层，可以适配各种策略模型（如扩散策略、VLA）和世界模型。
*   **确定性安全保障**：不依赖神经网络输出的概率置信度（通常不准），而是采用物理运动学误差作为硬性指标。
*   **动态自适应**：AHS 机制在带宽效率和控制实时性之间取得了良好的平衡。

### 8. 不足与局限
*   **仿真与现实差距（Sim-to-Real）**：所有实验均在 RLBench 仿真中完成，未在真实物理机器人上验证传感器噪声和动力学间隙的影响。
*   **安全反射过于生硬**：算法中的“安全停止”在仿真中是瞬时零速度，但在物理机器人上会导致机械应力，需要更复杂的阻抗控制或减速规划。
*   **世界模型依赖**：SPO 的性能高度依赖于世界模型的预测精度，如果世界模型在复杂接触下表现极差，SPO 会频繁退化为阻塞模式。
